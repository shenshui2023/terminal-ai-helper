# 项目结构说明

这个项目现在按“入口层、应用层、核心能力层、集成层、安装脚本、测试”来组织。规则类似常见后端框架：入口尽量薄，业务逻辑放到固定模块，外部集成单独隔离。

## 总览

```text
terminal-ai-helper/
  bin/                    对外 CLI 入口，保持 npm bin 兼容
  apps/                   可直接运行的本地应用
    cli/                  Node.js CLI 编排层
    powershell/           PowerShell 交互应用：profile、面板、托盘
  src/                    可复用核心模块
    ai/                   提示词和 AI 任务定义
    core/                 API、配置、渲染、剪贴板、缓存历史
    knowledge/            本地常用命令知识库
    server/               本地 HTTP helper server
  integrations/           外部软件和远端环境集成
    ssh/                  SSH 远端 bash 集成
    vscode/               VS Code 右键扩展
    windows-terminal/     Windows Terminal 集成说明
  scripts/                安装、配置、初始化脚本
    install/              所有安装脚本
  tests/                  自动化和 smoke test
    powershell/           PowerShell 面板、快捷键、托盘检查
  powershell/             旧路径兼容入口，内部转发到 apps/powershell
  remote/                 旧路径兼容入口，内部转发到 integrations/ssh
  docs/                   用户文档和维护文档
```

## 分层规则

| 层级 | 目录 | 职责 | 不应该做什么 |
| --- | --- | --- | --- |
| 入口层 | `bin/`、`powershell/`、`remote/` | 保持外部命令和旧路径兼容，只做转发 | 不放业务逻辑 |
| 应用层 | `apps/cli`、`apps/powershell` | 组合核心模块，处理 UI、快捷键、进程、窗口 | 不直接保存密钥，不写重复 AI 请求逻辑 |
| 核心层 | `src/core` | API 调用、配置读取、缓存历史、渲染输出、剪贴板 | 不依赖 PowerShell 面板或 VS Code |
| AI 层 | `src/ai` | prompt、输出 schema、任务规则 | 不处理窗口、文件安装、快捷键 |
| 知识层 | `src/knowledge` | 本地快速候选和常用命令说明 | 不调用 API |
| 服务层 | `src/server` | 本地 HTTP 服务，给 SSH 反向隧道和远端调用 | 不耦合具体终端快捷键 |
| 集成层 | `integrations/` | VS Code、Windows Terminal、SSH 等外部环境适配 | 不修改核心业务规则 |
| 脚本层 | `scripts/install` | 安装 profile、托盘启动项、VS Code 扩展、用户环境变量 | 不实现运行时功能 |
| 测试层 | `tests/` | 语法、快捷键、面板、托盘 smoke test | 不依赖真实用户窗口状态 |

## 主要调用链

### CLI

```text
bin/taih.js
  -> apps/cli/main.js
    -> src/core/config.js
    -> src/ai/prompts.js
    -> src/core/api.js
    -> src/core/render.js
    -> src/core/store.js
```

### PowerShell 快捷键

```text
powershell/taih-profile.ps1
  -> apps/powershell/profile.ps1
    -> bin/taih.js
    -> apps/powershell/panel.ps1
```

### 右侧面板

```text
apps/powershell/panel.ps1
  -> bin/taih.js --stream
    -> apps/cli/main.js
      -> src/core/api.js
```

### SSH 远端

```text
远端 integrations/ssh/taih-bash.sh
  -> ssh -R 反向隧道
    -> 本机 node bin/taih.js serve
      -> src/server/http-server.js
        -> apps/powershell/panel.ps1 或 src/core/api.js
```

### VS Code

```text
integrations/vscode/terminal-ai-helper/extension.js
  -> node bin/taih.js --stream
  -> 或 apps/powershell/panel.ps1
```

## 兼容入口

为了不破坏已经安装过的命令和快捷方式，保留这些旧路径：

| 旧路径 | 新实现 |
| --- | --- |
| `bin/taih.js` | 转发到 `apps/cli/main.js` |
| `powershell/taih-profile.ps1` | 转发到 `apps/powershell/profile.ps1` |
| `powershell/panel.ps1` | 转发到 `apps/powershell/panel.ps1` |
| `powershell/tray.ps1` | 转发到 `apps/powershell/tray.ps1` |
| `powershell/install-*.ps1` | 转发到 `scripts/install/*.ps1` |
| `powershell/test-panel.ps1` | 转发到 `tests/powershell/panel.ps1` |
| `remote/taih-bash.sh` | 转发到 `integrations/ssh/taih-bash.sh` |

后续文档和新代码优先使用新路径；旧路径只作为兼容层保留。

## 新功能放置规则

- 新增 CLI 子命令：优先改 `apps/cli/main.js`；如果命令变复杂，再拆到 `apps/cli/commands/`。
- 新增 AI 输出规则：放到 `src/ai/prompts.js`。
- 新增本地候选或常用命令说明：放到 `src/knowledge/local-help.js`。
- 新增 API、缓存、配置、渲染能力：放到 `src/core/` 对应模块。
- 新增 PowerShell UI：放到 `apps/powershell/`。
- 新增 VS Code、Windows Terminal、SSH 等外部集成：放到 `integrations/<目标>/`。
- 新增安装脚本：放到 `scripts/install/`。
- 新增测试：放到 `tests/`，并接入 `npm test` 或 `npm run check`。

## 测试命令

```powershell
npm run check
npm test
```

需要真实 API 回归时：

```powershell
npm run test:api
```

## 维护建议

- 不要在源码中写 API key，统一通过环境变量或用户配置读取。
- 不要让 `src/core` 反向依赖 `apps/`，核心层必须保持可复用。
- PowerShell UI 尽量只处理交互和窗口，不重复实现 AI 请求协议。
- 新增外部集成时先写在 `integrations/`，验证稳定后再考虑加入安装脚本。
- 旧兼容入口不要继续膨胀，只做一行转发。
