# 项目结构说明

terminal-ai-helper 按“入口层、应用层、核心能力层、集成层、安装脚本、测试”组织。目标是让每个能力有清楚边界：本机桌面捕获、PowerShell 命令行集成、SSH 远端增强、AI API 调用互不混在一起。

## 总览

```text
terminal-ai-helper/
  bin/                    对外 CLI 入口
  apps/                   可直接运行的本地应用
    cli/                  Node.js CLI 编排层
    powershell/           PowerShell 交互应用
      profile.ps1         本地 PowerShell 快捷键和命令行集成
      panel.ps1           管理面板
      complete-popup.ps1  智能补全候选框
      desktop-terminal.ps1 本机桌面终端文本捕获和写回
      tray.ps1            系统托盘和全局热键
      ssh-panel.ps1       本地 SSH 控制面板
  src/                    可复用核心模块
    ai/                   提示词和 AI 任务定义
    core/                 API、配置、渲染、剪贴板、缓存历史
    knowledge/            本地常用命令知识
    server/               本地 HTTP helper server
  integrations/           外部软件和远端环境适配
    ssh/                  可选 SSH 远端 bash/readline 增强
    vscode/               VS Code 右键扩展
    windows-terminal/     Windows Terminal 集成说明
  scripts/                安装、配置、初始化脚本
  tests/                  自动化和 smoke test
  powershell/             旧路径兼容入口，转发到 apps/powershell
  remote/                 旧路径兼容入口，转发到 integrations/ssh
  docs/                   用户文档和维护文档
```

## 分层规则

| 层级 | 目录 | 职责 |
| --- | --- | --- |
| 入口层 | `bin/`、`powershell/`、`remote/` | 保持外部命令和旧路径兼容，只做转发 |
| 应用层 | `apps/cli`、`apps/powershell` | 组合核心能力，处理 UI、快捷键、窗口、进程 |
| 桌面捕获层 | `apps/powershell/desktop-terminal.ps1` | 读取本机 Windows Terminal 可见文本，从最后一行提取命令，并把候选写回原窗口 |
| 核心层 | `src/core` | API 调用、配置读取、缓存历史、渲染输出、剪贴板 |
| AI 层 | `src/ai` | prompt、输出 schema、任务规则 |
| 知识层 | `src/knowledge` | 本地快速候选和常用命令说明 |
| 服务层 | `src/server` | 本地 HTTP 服务，给可选 SSH 反向隧道和远端调用使用 |
| 集成层 | `integrations/` | VS Code、Windows Terminal、SSH 等外部环境适配 |
| 脚本层 | `scripts/install` | 安装 profile、托盘启动项、VS Code 扩展、用户环境变量 |
| 测试层 | `tests/` | 语法、快捷键、面板、托盘、文本提取 smoke test |

## 主要调用链

### 本地 PowerShell 快捷键

```text
powershell/taih-profile.ps1
  -> apps/powershell/profile.ps1
    -> bin/taih.js
    -> apps/powershell/panel.ps1
    -> apps/powershell/complete-popup.ps1
```

### SSH 标签页本机直接读取当前输入

```text
powershell/tray.ps1
  -> apps/powershell/tray.ps1
    -> apps/powershell/desktop-terminal.ps1
      -> Windows UI Automation 读取当前 Windows Terminal 可见文本
      -> 从最后一行 prompt 提取命令
    -> apps/powershell/complete-popup.ps1 或 panel.ps1
    -> 选择补全后通过剪贴板/键盘写回原 Windows Terminal 窗口
```

这条链路不需要服务器安装项目，也不需要 `ssh -R`。它是 SSH 场景的默认推荐路径。

### 可选 SSH 远端 readline 增强

```text
远端 integrations/ssh/taih-bash.sh
  -> 读取 READLINE_LINE
  -> ssh -R 反向隧道
    -> 本机 node bin/taih.js serve
      -> src/server/http-server.js /complete-popup
        -> apps/powershell/complete-popup.ps1
  <- 写回远端 READLINE_LINE
```

这条链路只在本机 UI Automation 读不到当前输入，或确实需要远端 readline 结构化当前行时使用。

### 管理面板

```text
apps/powershell/panel.ps1
  -> bin/taih.js --stream
    -> apps/cli/main.js
      -> src/core/api.js
```

### 本地补全弹窗

```text
apps/powershell/complete-popup.ps1
  -> 本地候选立即显示
  -> node bin/taih.js complete --json --no-cache
    -> src/core/api.js
  -> 返回 JSON 给调用方
```

### 本地 SSH 控制面板

```text
powershell/ssh-panel.ps1
  -> apps/powershell/ssh-panel.ps1
    -> ssh <目标> -- bash -lc <命令>
    -> bin/taih.js explain/complete/fix/tools
```

## 新功能放置规则

- 新增 CLI 子命令：放到 `apps/cli/main.js`；复杂后再拆到 `apps/cli/commands/`。
- 新增 AI 输出规则：放到 `src/ai/prompts.js`。
- 新增本地候选或常用命令说明：放到 `src/knowledge/local-help.js` 或对应 UI 的本地候选函数。
- 新增 API、缓存、配置、渲染能力：放到 `src/core/`。
- 新增 PowerShell UI：放到 `apps/powershell/`。
- 新增本机桌面窗口读取/写回能力：放到 `apps/powershell/desktop-terminal.ps1`。
- 新增 VS Code、Windows Terminal、SSH 等外部集成：放到 `integrations/<目标>/`。
- 新增安装脚本：放到 `scripts/install/`。
- 新增测试：放到 `tests/`，并接入 `npm test` 或 `npm run check`。
