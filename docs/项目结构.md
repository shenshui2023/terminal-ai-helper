# 项目结构说明

terminal-ai-helper 现在按“核心能力”和“桌面交互”分层。核心原则是：AI、缓存、提示词、HTTP 服务不依赖 UI；桌面端只负责任务入口、快捷键、终端文本读取和窗口编排。

## 目录总览

```text
terminal-ai-helper/
  bin/
    taih.js                         CLI 入口，保持对外命令稳定

  apps/
    cli/                            Node CLI 编排层
    desktop/
      TerminalAi.Desktop/           C# 桌面常驻程序，负责桌面交互
    powershell/                     PowerShell 适配层和旧 UI

  src/
    ai/                             提示词、任务定义、输出规则
    core/                           API、配置、缓存、历史、渲染、剪贴板
    knowledge/                      本地常用命令和快速候选
    server/                         本地 HTTP helper server

  integrations/
    ssh/                            可选 SSH readline 集成
    vscode/                         VS Code 右键扩展
    windows-terminal/               Windows Terminal 集成说明

  scripts/
    desktop.ps1                     启动 TerminalAi.Desktop
    install/                        安装、开机启动、环境配置脚本

  tests/                            自动化检查和 smoke test
  powershell/                       旧路径兼容入口，转发到 apps/powershell
  remote/                           旧路径兼容入口，转发到 integrations/ssh
```

## 分层规则

| 层级 | 目录 | 职责 | 不应该做的事 |
| --- | --- | --- | --- |
| 核心能力 | `src/core` | API 请求、缓存、历史、配置、渲染 | 不直接创建窗口、不注册快捷键 |
| AI 任务 | `src/ai` | prompt、schema、输出格式规则 | 不读写 UI 状态 |
| 本地知识 | `src/knowledge` | 高频命令本地候选和说明 | 不调用中转站 |
| CLI 编排 | `apps/cli` | 参数解析、调用核心能力、输出到 stdout | 不控制桌面窗口 |
| 桌面端 | `apps/desktop/TerminalAi.Desktop` | 托盘、全局快捷键、终端文本读取、写回终端、启动本地服务 | 不直接实现 AI 请求细节 |
| PowerShell 适配 | `apps/powershell` | 现有 profile、面板、补全弹窗的兼容实现 | 不再承载新的核心业务 |
| 外部集成 | `integrations` | SSH、VS Code、Windows Terminal 适配 | 不复制核心逻辑 |
| 安装脚本 | `scripts/install` | 写 profile、开机启动、安装扩展 | 不实现业务功能 |

## TerminalAi.Desktop 设计

`TerminalAi.Desktop` 是新的桌面交互入口，当前采用 C# WinForms：

```text
TerminalAi.Desktop
  Infrastructure/
    ProjectRootResolver.cs          定位项目根目录
    ProcessLauncher.cs              统一启动 node / powershell / dotnet
    NativeMethods.cs                Win32 P/Invoke
    GlobalHotKeyWindow.cs           全局快捷键消息窗口
    TerminalCaptureService.cs       UI Automation 读取终端文本、SendKeys 写回

  Services/
    LocalServerManager.cs           管理本地 helper server
    PowerShellBridge.cs             临时桥接旧 PowerShell 面板和补全弹窗
    TerminalAiDesktopActions.cs     桌面动作编排

  Ui/
    TerminalAiApplicationContext.cs 托盘菜单、热键注册、生命周期
```

当前 C# 桌面端会复用 `apps/powershell/panel.ps1` 和 `apps/powershell/complete-popup.ps1` 作为过渡 UI。后续如果要做原生 C# 面板，只需要替换 `PowerShellBridge`，不要改 `src/core`、`src/ai` 和 `apps/cli`。

## 推荐调用链

### 本地终端补全

```text
Ctrl+Alt+P
  -> TerminalAi.Desktop
    -> TerminalCaptureService 读取当前 Windows Terminal 文本
    -> PowerShellBridge 打开补全候选框
    -> apps/powershell/complete-popup.ps1
      -> bin/taih.js complete
        -> src/knowledge 本地候选
        -> src/core/api 中转站候选
    -> TerminalCaptureService 写回原终端
```

### 本地终端解释

```text
Ctrl+Alt+E
  -> TerminalAi.Desktop
    -> TerminalCaptureService 读取当前命令
    -> PowerShellBridge 打开右侧面板
    -> apps/powershell/panel.ps1
      -> bin/taih.js explain --stream
        -> src/core/cache / src/core/api
```

### 可选 SSH readline 模式

```text
远端 integrations/ssh/taih-bash.sh
  -> 本机 node bin/taih.js serve
    -> src/server/http-server.js
      -> 本机 PowerShell 弹窗或面板
```

SSH 默认优先用 `TerminalAi.Desktop` 直接读取本机 Windows Terminal 的可见文本。只有 UI Automation 读不到当前行，或者确实需要远端 readline 精确当前行时，才使用 SSH 反向隧道。

## 新功能放置规则

- 新增 API 参数、模型策略、代理、超时、缓存：放到 `src/core`。
- 新增提示词、输出格式、工具集规则：放到 `src/ai`。
- 新增本地高频命令候选：放到 `src/knowledge`。
- 新增 CLI 子命令：先放 `apps/cli/main.js`，复杂后拆到 `apps/cli/commands/`。
- 新增桌面快捷键、托盘菜单、窗口贴靠、终端读取写回：放到 `apps/desktop/TerminalAi.Desktop`。
- 新增旧 PowerShell 兼容 UI：放到 `apps/powershell`，但不要再把核心逻辑写进去。
- 新增 VS Code、SSH、Windows Terminal 集成：放到 `integrations/<target>`。
- 新增安装入口：放到 `scripts/install`。
- 新增测试：放到 `tests/`，并接入 `npm test` 或 `npm run check`。

## 常用命令

```powershell
npm run check
npm test

# 启动 C# 桌面端
powershell -ExecutionPolicy Bypass -File .\scripts\desktop.ps1

# 安装 C# 桌面端开机启动
powershell -ExecutionPolicy Bypass -File .\scripts\install\desktop-startup.ps1 -Build
```
