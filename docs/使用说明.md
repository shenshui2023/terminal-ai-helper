# terminal-ai-helper 使用说明

terminal-ai-helper 是一个面向终端命令的桌面 AI 助手。它可以解释命令、补全命令、诊断报错、生成常用工具菜单，并支持本地 PowerShell、Windows Terminal、系统托盘、VS Code 右键和 SSH 场景。

## 快速开始

```powershell
cd E:\3.13-aliyun-codex\5.2\terminal-ai-helper
powershell -ExecutionPolicy Bypass -File .\scripts\install\all.ps1
. .\powershell\taih-profile.ps1
```

常用入口：

| 场景 | 推荐用法 |
| --- | --- |
| 本地 PowerShell 补全 | `F4` 或 `Alt+P` |
| 本地 PowerShell 解释 | `Alt+/` 或 `F2` |
| 打开管理面板 | `Alt+?`、`F3` 或 `taih-panel` |
| SSH 标签页当前输入补全 | 启动托盘后按 `Ctrl+Alt+P` |
| SSH 标签页当前输入解释 | 启动托盘后按 `Ctrl+Alt+E` |
| 分析 SSH 标签页已显示文本 | 鼠标选中后按 `Ctrl+Alt+/` 或 `Ctrl+Alt+F` |
| SSH 远端控制面板 | `taih-ssh-panel` |
| VS Code 选中文本 | 右键使用“终端 AI 助手” |

## 管理面板

```powershell
taih-panel
```

管理面板支持：

- `explain（解析命令）`：说明命令作用、常用参数、示例和风险。
- `fix（诊断报错）`：分析报错原因，给出修复步骤和验证命令。
- `complete（补全命令）`：根据当前输入生成候选命令。
- `tools（工具菜单）`：按工具集生成常用基础命令菜单。
- 配置模型、接口地址、超时时间、工具集、输出格式和自定义提示词。
- 查看历史记录、清缓存、复制输出。

## 本地 PowerShell 智能补全

在 PowerShell 输入命令前缀，例如：

```powershell
kube get svc
```

按 `F4` 或 `Alt+P` 会弹出候选框。`Ctrl+Space` 在很多中文输入法里是中英文切换键，所以不作为主推荐键。

- 本地候选立即显示，`git`、`ssh`、`docker`、`npm`、`python`、`java`、`adb`、`kube/kubectl` 等常见命令不用先等 API。
- AI 候选会在后台追加，不要求所有命令都预存在本地。
- 候选可以在下方输入框直接修改，按 `Enter` 插回当前命令行。
- `解释` 按钮、`Ctrl+E` 或 `F1` 会把当前候选送到管理面板解释。
- `Esc` 关闭候选框。

## SSH 标签页：本机直接读取当前输入

这是 SSH 场景的推荐方式。它不需要服务器安装项目，不需要 `ssh -R`，也不需要把 API key 放到服务器。它直接读取你本机 Windows Terminal 当前标签页的可见文本，从最后一行提取正在输入的命令。

启动托盘：

```powershell
powershell -ExecutionPolicy Bypass -File E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\tray.ps1
```

在 SSH 标签页里输入命令前缀，例如：

```bash
kube get svc
```

然后使用：

| 快捷键 | 功能 |
| --- | --- |
| `Ctrl+Alt+P` | 读取当前终端输入，弹出本机补全候选框，选择后写回当前 SSH 命令行 |
| `Ctrl+Alt+E` | 读取当前终端输入，打开本机管理面板解释 |
| `Ctrl+Alt+/` | 解释鼠标选中的终端文本 |
| `Ctrl+Alt+F` | 诊断鼠标选中的终端文本 |

如果当前终端控件没有暴露可访问文本，`Ctrl+Alt+P/E` 可能读不到输入。此时可以鼠标选中文本后按 `Ctrl+Alt+/`，或者使用远端 readline 增强。

## SSH 远端控制面板

如果你想在一个独立桌面窗口里执行远端命令和查看 AI 输出：

```powershell
taih-ssh-panel
```

默认连接 `root@us-vpn`。也可以指定目标：

```powershell
powershell -ExecutionPolicy Bypass -File E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\ssh-panel.ps1 -Target root@43.112.115.205
```

这个方式也不需要在服务器安装 terminal-ai-helper，AI 请求仍然由本机发出。

## 可选：远端 readline 增强

只有在本机 UI Automation 读不到当前输入，或你明确需要远端 bash/readline 级别的当前行时，才需要反向隧道。

本机：

```powershell
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js serve --port 17888 --replace
ssh -R 17888:127.0.0.1:17888 <用户名>@<主机>
```

远端：

```bash
source /tmp/taih-bash.sh
```

远端可用 `F4` 或 `Alt+P` 打开本机补全框。

## 故障处理

快捷键没反应：

```powershell
taih-keys
taih-what-key
```

面板卡住：

```powershell
taih-panel-reset
taih-panel
```

API 不通：

```powershell
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js doctor
```
