# terminal-ai-helper 使用说明

terminal-ai-helper 是一个面向终端命令的 AI 助手。它可以解释命令、补全命令、分析报错，并提供 PowerShell 快捷键、右侧面板、系统托盘、SSH 远端 shell、VS Code 右键等入口。

## 快速开始

免安装启动：

```powershell
E:\3.13-aliyun-codex\5.2\terminal-ai-helper\terminal-ai-helper.cmd
```

在当前 PowerShell 会话临时加载：

```powershell
. E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\taih-profile.ps1
```

一键安装本地入口：

```powershell
powershell -ExecutionPolicy Bypass -File E:\3.13-aliyun-codex\5.2\terminal-ai-helper\scripts\install\all.ps1
```

## 常用入口

| 场景 | 推荐入口 |
| --- | --- |
| 本地 PowerShell 当前命令 | `Alt+/`、`F4`、`Alt+F` |
| 打开右侧管理面板 | `Alt+?` 或 `taih-panel` |
| SSH 标签页里已经显示的文本 | 鼠标选中后按 `Ctrl+Alt+/` 或 `Ctrl+Alt+F` |
| SSH 远端当前正在编辑的命令 | SSH 反向隧道 + `integrations/ssh/taih-bash.sh` |
| VS Code 里选中的命令或日志 | 右键“终端 AI 助手” |

## 本机能看到 SSH 文本吗

能。你通过 SSH 操控服务器时，命令和输出确实显示在本机 Windows Terminal 里，所以本机助手可以分析这些文本。

关键区别是：

- Windows Terminal UI 能显示、选中、复制 SSH 文本。
- 服务器 bash/readline 不能读取 Windows Terminal 的鼠标选区。
- 本机 PowerShell 的 PSReadLine 也不能在 SSH 会话里直接读取远端正在编辑的命令，因为输入已经交给 ssh 进程和远端 shell。

因此有两个方案：

1. 分析桌面上已经看到的文本：用鼠标选中，按托盘全局热键。
2. 不选中文本，直接补全远端当前命令行：用 SSH 反向隧道和远端 bash 集成。

## 托盘全局选区

启动托盘：

```powershell
powershell -ExecutionPolicy Bypass -File E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\tray.ps1
```

用法：

- 在 Windows Terminal 的 SSH 标签页里选中命令、报错或日志。
- `Ctrl+Alt+/`：解释选中文本。
- `Ctrl+Alt+F`：诊断选中文本。

注意：热键会发送一次 `Ctrl+C` 来复制选区。没有选中文本时，远端 shell 可能会把 `Ctrl+C` 当作中断。

## SSH 远端命令行集成

本机启动 helper server：

```powershell
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js serve --port 17888
```

用反向隧道连接服务器：

```powershell
ssh -R 17888:127.0.0.1:17888 <用户名>@<主机>
```

远端加载 bash 集成：

```bash
source /path/to/terminal-ai-helper/integrations/ssh/taih-bash.sh
```

远端常用命令：

```bash
taih explain 'systemctl status <服务名>'
journalctl -u <服务名> -n 80 --no-pager | taih fix
taih-panel explain 'systemctl status <服务名>'
journalctl -u <服务名> -n 80 --no-pager | taih-panel fix
```

## 面板和输出规则

打开面板：

```powershell
taih-panel
```

面板支持：

- `explain（解析命令）`
- `fix（诊断报错）`
- `complete（补全命令）`
- 输出风格：`brief`、`standard`、`examples`、`custom`
- 自定义规则，例如“只输出作用、常用参数、两个示例”
- 历史记录、缓存清理、剪贴板读取
