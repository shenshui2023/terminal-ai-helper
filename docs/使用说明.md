# terminal-ai-helper 使用说明

terminal-ai-helper 是本机桌面 AI 命令助手。SSH 场景默认不要求服务器安装脚本，也不要求反向端口；它直接读取你本机 Windows Terminal 里当前可见的命令文本，再用本机补全、解释、诊断和写回能力处理。

## 快速开始

```powershell
cd E:\3.13-aliyun-codex\5.2\terminal-ai-helper
powershell -ExecutionPolicy Bypass -File .\scripts\install\all.ps1
```

本地 PowerShell 快捷键：

```powershell
. .\powershell\taih-profile.ps1
```

本机托盘常驻程序：

```powershell
powershell -ExecutionPolicy Bypass -File E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\tray.ps1
```

## 推荐入口

| 场景 | 推荐方式 |
| --- | --- |
| 本地 PowerShell 补全 | `F4` 或 `Alt+P` |
| 本地 PowerShell 解释 | `Alt+/` 或 `F2` |
| 打开管理面板 | `Alt+?`、`F3` 或 `taih-panel` |
| SSH 标签页当前输入补全 | 启动托盘后按 `Ctrl+Alt+P` |
| SSH 标签页当前输入解释 | 启动托盘后按 `Ctrl+Alt+E` |
| 分析 SSH 标签页已显示文本 | 鼠标选中文本后按 `Ctrl+Alt+/` 或 `Ctrl+Alt+F` |
| 独立 SSH 控制面板 | `taih-ssh-panel` |
| VS Code 选中文本 | 右键使用“终端 AI 助手” |

## SSH 标签页：本机直接处理

这是 SSH 场景的主路径。

它的工作方式是：

1. 你在本机 Windows Terminal 里连接服务器并输入命令。
2. 托盘热键读取当前 Windows Terminal 可见文本。
3. 程序从最后一行提取当前正在输入的命令。
4. 本机弹出补全框或管理面板。
5. 选择补全后，本机把结果写回当前 SSH 命令行。

服务器不需要安装 terminal-ai-helper，不需要 `source` 脚本，不需要 `ssh -R`，API key 也不会放到服务器。

常用热键：

| 快捷键 | 功能 |
| --- | --- |
| `Ctrl+Alt+P` | 读取当前 SSH 标签页输入，打开本机补全框，选择后写回 |
| `Ctrl+Alt+E` | 读取当前 SSH 标签页输入，打开本机解释面板 |
| `Ctrl+Alt+/` | 解释鼠标选中的终端文本 |
| `Ctrl+Alt+F` | 诊断鼠标选中的终端文本 |

如果 `Ctrl+Alt+P/E` 读不到当前输入，通常是当前终端控件没有向 Windows 辅助功能接口暴露文本。优先用鼠标选中命令或报错后按 `Ctrl+Alt+/` 或 `Ctrl+Alt+F`。

## 本地 PowerShell 补全

在 PowerShell 输入命令前缀，例如：

```powershell
kube get svc
```

按 `F4` 或 `Alt+P` 打开补全框。`Ctrl+Space` 在很多中文输入法里是中英文切换键，所以不是主推荐键。

补全框能力：

- 本地候选立即显示。
- AI 候选在后台追加。
- 下方输入框可以直接修改候选命令。
- `方向提示` 可以写模糊目标，例如“偏网络排查”“偏 k8s service”“偏 Linux CPU 信息”。
- `刷新AI` 会跳过旧缓存重新请求 AI，并把新候选写回缓存。
- `Enter` 插回当前命令行。
- `解释`、`Ctrl+E` 或 `F1` 会把候选送到管理面板解释。
- `Esc` 关闭。

## 管理面板

```powershell
taih-panel
```

支持：

- `explain（解析命令）`
- `fix（诊断报错）`
- `complete（补全命令）`
- `tools（工具菜单）`
- 历史记录
- 模型、接口地址、工具集和输出格式配置
- 自定义提示词规则
- 缓存统计与清理

## SSH 控制面板

如果你想在独立桌面窗口里操作远端服务器，可以用：

```powershell
taih-ssh-panel
```

这个面板也是本机功能：它通过本机 `ssh <目标> -- bash -lc <命令>` 执行远端命令，AI 请求仍然由本机发出。

## 高级备用：远端 readline

远端 bash/readline 脚本不是默认路径。只有在本机无法读取 Windows Terminal 可见文本，并且你明确需要远端 readline 当前行时才考虑。

日常 SSH 标签页补全和解释，请优先使用本机托盘：

```powershell
powershell -ExecutionPolicy Bypass -File E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\tray.ps1
```

## 故障处理

快捷键没有反应：

```powershell
taih-keys
taih-what-key
```

面板卡住：

```powershell
taih-panel-reset
taih-panel
```

API 配置检查：

```powershell
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js doctor
```
