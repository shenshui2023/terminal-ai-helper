# terminal-ai-helper 使用说明

terminal-ai-helper 是一个面向终端命令的 AI 助手。它可以解释命令、补全命令、分析报错，并提供 PowerShell 面板、系统托盘、SSH 远端 shell、VS Code 选中文本等入口。

## 快速开始

加载 PowerShell 集成：

```powershell
. E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\taih-profile.ps1
```

常用快捷键：

| 推荐快捷键 | 兼容写法 | 功能 |
| --- | --- | --- |
| `Alt+/` | 无 | 在终端中直接解释当前命令 |
| `Alt+?` | `Alt+Shift+/` | 打开管理面板 |
| `Ctrl+Space` | 无 | 在光标下方打开智能补全候选框，选择或编辑后插入 |
| `Alt+C` | `Alt+Shift+C` | 复制 AI 补全 |
| `Alt+F` | `Alt+Shift+F` | 诊断当前命令或报错 |

说明：Windows Terminal、PowerShell、输入法和键盘布局会影响组合键名称。很多环境会把 `Alt+Shift+C` 识别成 `Alt+C`，把 `Alt+Shift+/` 识别成 `Alt+?`，所以请优先使用上表“推荐快捷键”。

## 管理面板

管理面板现在以独立 PowerShell 进程运行。当前终端只负责唤起面板，面板打开后终端可以继续输入和执行命令，不会再被窗口占住。

面板会尽量无缝拼接到当前终端右侧；如果右侧空间不足，会自动贴到终端另一侧。面板会跟随当前终端移动，终端窗口关闭时面板也会自动关闭。同一个终端重复按快捷键会复用已有面板并更新内容。窗口风格尽量接近终端：黑底、Consolas 字体、紧凑输出。面板包含：

- 输入框：可以手动输入命令或报错。
- 模式选择：`explain（解析命令）`、`fix（诊断报错）`、`complete（补全命令）`。
- 输出风格：`brief（简洁）`、`standard（标准）`、`examples（示例优先）`、`custom（按规则）`。
- 规则框：可以写自定义输出格式或提示词。
- 输出区：支持鼠标滚轮、选择文本、复制。
- `读剪贴板`：读取剪贴板并执行。
- `复制`：复制输出内容。
- `清空`：清空输出内容。

输出风格建议：

| 风格 | 适合场景 |
| --- | --- |
| `brief（简洁）` | 日常查命令，输出尽量短 |
| `standard（标准）` | 常规说明、风险和下一步 |
| `examples（示例优先）` | 重点看可复制示例 |
| `custom（按规则）` | 按规则框里的要求输出 |

默认输出会控制空行和缩进：段落之间留一个空行，列表和示例用两个空格缩进。规则框示例：

```text
只输出三段：作用、常用参数、两个示例。
不要输出长表格。
示例必须说明作用，并使用 <文件路径> 这类占位符。
```

如果想分析终端历史输出，先在终端里选中内容并复制，然后点击面板里的 `读剪贴板`。

面板左侧窄栏是历史记录，点击历史项可以回填命令；如果历史中保存了输出，会直接显示旧结果。历史区默认较窄，优先把空间留给输出区。同一个终端再次使用 `Alt+?`、`Alt+Shift+/` 或 `taih-panel` 时，默认复用已经打开的面板。

`Ctrl+Space` 会在当前终端光标附近打开智能补全候选框。候选框先显示本地快速建议，同时后台请求 AI 候选；你可以选择候选，也可以在下方输入框直接修改完整命令，然后按 Enter 插回当前命令行。

缓存说明：

- `git`、`ssh`、`docker` 这类常见命令在 `brief explain` 下会先走本地快速解释，第一次也不用等 API。
- 其他命令的流式结果会写入本地缓存；相同命令和相同输出规则再次查询会直接显示缓存。

命令行也可以使用同一套输出控制：

```powershell
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js explain --style brief -- "git status"
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js explain --style custom --instructions-file .\my-rules.txt -- "ssh <用户名>@<主机>"
```

## 系统托盘

启动托盘常驻：

```powershell
powershell -ExecutionPolicy Bypass -File E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\tray.ps1
```

安装为开机自启：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\install-tray-startup.ps1
```

托盘菜单支持：

- 打开管理面板
- 解释剪贴板
- 诊断剪贴板
- 启动或停止 SSH helper server

## SSH 中使用

进入 SSH 后，本地 PowerShell 的快捷键不会再接管命令行，因为当前命令由远端 shell 处理。

推荐方案：

1. 本地启动服务：

```powershell
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js serve --port 17888
```

2. 使用反向隧道连接服务器：

```powershell
ssh -R 17888:127.0.0.1:17888 <用户名>@<主机>
```

3. 在远端加载 bash 集成：

```bash
source /path/to/terminal-ai-helper/remote/taih-bash.sh
```

远端快捷键：

| 快捷键 | 功能 |
| --- | --- |
| `Alt+/` | 解释远端当前命令 |
| `Alt+?` | 把远端当前命令发送到本机管理面板 |
| `Ctrl+Space` | 补全远端当前命令 |
| `Alt+F` | 诊断远端当前命令 |

远端直接分析文本：

```bash
taih explain 'systemctl status <服务名>'
taih fix 'Permission denied (publickey)'
journalctl -u <服务名> -n 80 --no-pager | taih fix
cat <日志文件> | taih-panel fix
taih-panel explain 'systemctl status <服务名>'
```

说明：远端 shell 不能直接读取你在 Windows Terminal 里用鼠标选中的文本，因为选区属于本机终端 UI，不属于服务器 bash/readline。更稳的方案是用管道、参数或 `taih-panel` 把文本发回本机，本机保留 API key 并负责展示管理面板。

如果远端快捷键没反应，先在服务器里运行：

```bash
taih-keys
```

常见原因：

- 当前不是交互式 `bash`，`bind -x` 不生效；可以改用 `taih ...` 或 `taih-panel ...` 命令。
- 没有在当前 SSH 会话里执行 `source /path/to/terminal-ai-helper/remote/taih-bash.sh`。
- Windows Terminal、输入法或远端程序抢走了 `Ctrl+Space` / `Alt+F`。远端脚本同时支持 `Alt+F` 和 `Alt+f`，也可以直接用命令兜底。
- 本机没有启动 `node ...\taih.js serve --port 17888`，或 SSH 没带 `-R 17888:127.0.0.1:17888`。

## VS Code 集成

安装本地 VS Code 扩展：

```powershell
powershell -ExecutionPolicy Bypass -File E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\install-vscode-extension.ps1
```

安装后可以：

- 在编辑器中选中文本，右键选择“终端 AI 助手：解释选中文本”。
- 右键选择“终端 AI 助手：诊断选中文本”。
- 从命令面板运行“终端 AI 助手：解释剪贴板”。

## Windows Terminal 说明

Windows Terminal 目前不提供稳定接口让外部脚本直接读取右键菜单中的选中文本。因此推荐使用：

1. 选中文本后复制。
2. 使用托盘菜单“解释剪贴板”或管理面板“读剪贴板”。

这是目前最稳的工作流。

## 缓存和历史

重复解释同一条命令时会使用本地缓存，加快响应速度。

清理缓存：

```powershell
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js cache clear
```

查看历史：

```powershell
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js history
```
