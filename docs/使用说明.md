# terminal-ai-helper 使用说明

terminal-ai-helper 是一个面向终端命令的 AI 助手。它支持解释命令、补全命令、诊断报错、生成常用工具菜单，并提供 PowerShell 快捷键、桌面管理面板、系统托盘、VS Code 右键、SSH 远端集成等入口。

## 快速开始

```powershell
cd E:\3.13-aliyun-codex\5.2\terminal-ai-helper
powershell -ExecutionPolicy Bypass -File .\scripts\install\all.ps1
. .\powershell\taih-profile.ps1
```

常用入口：

| 场景 | 用法 |
| --- | --- |
| 解释本地当前命令 | `Alt+/` |
| 打开管理面板 | `Alt+?` 或 `taih-panel` |
| 本地智能补全 | `Ctrl+Space`、`Ctrl+Shift+Space`、`F4` |
| SSH 远端控制面板 | `taih-ssh-panel` |
| SSH 标签页已显示文本 | 鼠标选中后按托盘全局热键 `Ctrl+Alt+/` 或 `Ctrl+Alt+F` |
| SSH 远端当前正在编辑的命令 | helper server + SSH 反向隧道 + `integrations/ssh/taih-bash.sh` |
| 常用工具菜单 | 管理面板选择 `tools（工具菜单）`，再选择工具集 |
| VS Code 选中文本 | 右键使用“终端 AI 助手” |

## 管理面板

```powershell
taih-panel
```

管理面板支持：

- `explain（解析命令）`：说明命令作用、常用参数、示例和风险。
- `fix（诊断报错）`：分析报错原因，给出修复步骤和验证命令。
- `complete（补全命令）`：根据当前输入生成候选命令。
- `tools（工具菜单）`：按工具集生成常用基础命令菜单。
- 配置模型、接口地址、超时时间、工具集、输出格式和自定义提示词。
- 查看历史记录、清缓存、复制输出。

## 本地补全

在 PowerShell 输入命令前缀，例如：

```powershell
kube get svc
```

按 `Ctrl+Space` 或 `F4` 后会弹出候选框：

- 本地候选立即显示，`git`、`ssh`、`docker`、`npm`、`python`、`java`、`adb`、`kube/kubectl` 等常见命令不需要先等 API。
- AI 候选会在后台追加，不要求所有命令都预存在本地。
- 候选可以在下方输入框直接修改，按 `Enter` 插回当前命令行。
- `解释` 按钮、`Ctrl+E` 或 `F1` 会把当前候选送到管理面板解释。
- `Esc` 关闭候选框。

## SSH 远端使用

### 方式一：本地 SSH 远端控制面板

如果你想在自己的桌面窗口里直接控制远端服务器，用：

```powershell
taih-ssh-panel
```

也可以指定目标：

```powershell
powershell -ExecutionPolicy Bypass -File E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\ssh-panel.ps1 -Target root@43.112.115.205
```

这个方式不需要把 API key 放到服务器，也不需要在服务器安装项目。它由本机通过 `ssh <目标> -- bash -lc <命令>` 执行远端命令，AI 请求仍然由本机发出。

### 方式二：分析 SSH 标签页里已经显示的文本

如果只是想分析屏幕上已经显示出来的远端命令、报错或日志，推荐用托盘全局选区：

```powershell
powershell -ExecutionPolicy Bypass -File E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\tray.ps1
```

然后在 Windows Terminal 的 SSH 标签页里选中文本：

- `Ctrl+Alt+/`：解释选中文本。
- `Ctrl+Alt+F`：诊断选中文本。

注意：没有选中文本时，`Ctrl+C` 可能会被远端 shell 当作中断。

### 方式三：远端当前命令行补全

如果你想在 SSH 会话里“不选中文本，直接读取远端当前正在编辑的命令行”，需要让远端 bash/readline 把当前行通过反向隧道发回本机。

本机启动 helper server：

```powershell
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js serve --port 17888
```

如果提示端口已占用，通常表示 helper server 已经在运行，可以继续下一步。

连接服务器时打开反向隧道：

```powershell
ssh -R 17888:127.0.0.1:17888 <用户名>@<主机>
```

远端加载 bash 集成：

```bash
source /path/to/terminal-ai-helper/integrations/ssh/taih-bash.sh
```

远端快捷键：

| 快捷键 | 功能 |
| --- | --- |
| `Ctrl+Space` | 打开本机桌面智能补全候选框，选择后写回远端当前命令行 |
| `Alt+/` | 解释远端当前命令行 |
| `Alt+F` | 诊断远端当前命令行 |
| `Alt+?` | 打开本机管理面板 |
| `Alt+T` | 打开本机工具菜单 |

远端命令：

```bash
taih explain 'systemctl status <服务名>'
taih complete 'kubectl get svc'
taih-complete-popup 'kubectl get svc'
taih-panel explain 'kubectl get svc -A'
journalctl -u <服务名> -n 80 --no-pager | taih fix
taih-tools
taih-tools linux k8s docker
```

`taih-tools linux k8s docker` 会设置当前远端会话的工具集，后续解释、补全和工具菜单都会优先围绕这些工具。

## 测试

```powershell
npm run check
npm test
npm run test:api
powershell -NoProfile -ExecutionPolicy Bypass -File .\apps\powershell\complete-popup.ps1 -Prefix "kube get svc" -NoDialog
```

远端链路可以用反向隧道验证：

```powershell
node .\bin\taih.js serve --port 17888
ssh -R 17888:127.0.0.1:17888 root@us-vpn
```

远端执行：

```bash
curl -sS http://127.0.0.1:17888/health
taih-complete-popup 'kube get svc'
```

## 故障处理

快捷键没反应：

```powershell
taih-keys
taih-what-key
```

管理面板卡住：

```powershell
taih-panel-reset
taih-panel
```

API 不通：

```powershell
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js doctor
```
