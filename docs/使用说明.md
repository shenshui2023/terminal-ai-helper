# terminal-ai-helper 使用说明

terminal-ai-helper 是一个面向终端命令的 AI 助手。它可以解释命令、补全命令、分析报错，并提供 PowerShell 面板、系统托盘、SSH 远端 shell、VS Code 选中文本等入口。

## 快速开始

加载 PowerShell 集成：

```powershell
. E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\taih-profile.ps1
```

常用快捷键：

| 推荐快捷键 | 兼容写法 | 功能 |
| --- | --- | --- |
| `Alt+/` | 无 | 在终端中直接解释当前命令 |
| `Alt+?` | `Alt+Shift+/` | 打开管理面板 |
| `Ctrl+Space` | 无 | AI 补全并插入到当前光标 |
| `Alt+C` | `Alt+Shift+C` | 复制 AI 补全 |
| `Alt+F` | `Alt+Shift+F` | 诊断当前命令或报错 |

说明：Windows Terminal、PowerShell、输入法和键盘布局会影响组合键名称。很多环境会把 `Alt+Shift+C` 识别成 `Alt+C`，把 `Alt+Shift+/` 识别成 `Alt+?`，所以请优先使用上表“推荐快捷键”。

## 管理面板

管理面板现在以独立 PowerShell 进程运行。当前终端只负责唤起面板，面板打开后终端可以继续输入和执行命令，不会再被窗口占住。

面板会尽量拼接到当前终端左侧；如果左侧空间不足，会自动贴到终端另一侧。窗口风格尽量接近终端：黑底、Consolas 字体、紧凑输出。面板包含：

- 输入框：可以手动输入命令或报错。
- 模式选择：`explain`、`fix`、`complete`。
- 输出风格：`brief`、`standard`、`examples`、`custom`。
- 规则框：可以写自定义输出格式或提示词。
- 输出区：支持鼠标滚轮、选择文本、复制。
- `读剪贴板`：读取剪贴板并执行。
- `复制`：复制输出内容。
- `清空`：清空输出内容。

输出风格建议：

| 风格 | 适合场景 |
| --- | --- |
| `brief` | 日常查命令，输出尽量短 |
| `standard` | 常规说明、风险和下一步 |
| `examples` | 重点看可复制示例 |
| `custom` | 按规则框里的要求输出 |

规则框示例：

```text
只输出三段：作用、常用参数、两个示例。
不要输出长表格。
示例必须说明作用，并使用 <文件路径> 这类占位符。
```

如果想分析终端历史输出，先在终端里选中内容并复制，然后点击面板里的 `读剪贴板`。

面板左侧会显示最近历史，点击历史项可以回填命令；如果历史中保存了输出，会直接显示旧结果。

缓存说明：

- `git`、`ssh`、`docker` 这类常见命令在 `brief explain` 下会先走本地快速解释，第一次也不用等 API。
- 其他命令的流式结果会写入本地缓存；相同命令和相同输出规则再次查询会直接显示缓存。

命令行也可以使用同一套输出控制：

```powershell
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js explain --style brief -- "git status"
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js explain --style custom --instructions-file .\my-rules.txt -- "ssh <用户名>@<主机>"
```

## 系统托盘

启动托盘常驻：

```powershell
powershell -ExecutionPolicy Bypass -File E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\tray.ps1
```

安装为开机自启：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\install-tray-startup.ps1
```

托盘菜单支持：

- 打开管理面板
- 解释剪贴板
- 诊断剪贴板
- 启动或停止 SSH helper server

## SSH 中使用

进入 SSH 后，本地 PowerShell 的快捷键不会再接管命令行，因为当前命令由远端 shell 处理。

推荐方案：

1. 本地启动服务：

```powershell
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js serve --port 17888
```

2. 使用反向隧道连接服务器：

```powershell
ssh -R 17888:127.0.0.1:17888 <用户名>@<主机>
```

3. 在远端加载 bash 集成：

```bash
source /path/to/terminal-ai-helper/remote/taih-bash.sh
```

远端快捷键：

| 快捷键 | 功能 |
| --- | --- |
| `Alt+/` | 解释远端当前命令 |
| `Ctrl+Space` | 补全远端当前命令 |
| `Alt+F` | 诊断远端当前命令 |

## VS Code 集成

安装本地 VS Code 扩展：

```powershell
powershell -ExecutionPolicy Bypass -File E:\3.13-aliyun-codex\5.2\terminal-ai-helper\powershell\install-vscode-extension.ps1
```

安装后可以：

- 在编辑器中选中文本，右键选择“终端 AI 助手：解释选中文本”。
- 右键选择“终端 AI 助手：诊断选中文本”。
- 从命令面板运行“终端 AI 助手：解释剪贴板”。

## Windows Terminal 说明

Windows Terminal 目前不提供稳定接口让外部脚本直接读取右键菜单中的选中文本。因此推荐使用：

1. 选中文本后复制。
2. 使用托盘菜单“解释剪贴板”或管理面板“读剪贴板”。

这是目前最稳的工作流。

## 缓存和历史

重复解释同一条命令时会使用本地缓存，加快响应速度。

清理缓存：

```powershell
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js cache clear
```

查看历史：

```powershell
node E:\3.13-aliyun-codex\5.2\terminal-ai-helper\bin\taih.js history
```
