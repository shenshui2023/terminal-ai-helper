# 命令一级缓存

`src/knowledge/command-cache.json` 是项目内置命令知识库；`%USERPROFILE%\.terminal-ai-helper\command-cache.user.json` 是用户自定义命令库。

补全窗口、解释面板、工具菜单和桌面命令管理页都会读取同一套命令缓存。常规命令会先从本地立即显示，AI 只负责补充、纠错和从文档中提取新命令。

## 覆盖范围

- Windows / PowerShell：`Get-Service <服务名>`、`Get-Process`、`Test-NetConnection <主机> -Port <端口>`、`winget install <包名>` 等。
- Linux / systemd：`systemctl status <服务名>`、`journalctl -u <服务名> -f`、`ss -lntp`、`df -h` 等。
- Kubernetes：`kubectl get svc -A`、`kubectl describe pod <Pod名> -n <命名空间>` 等。
- Docker / MySQL / Git / SSH / npm / Python / Java / ADB：覆盖查看、诊断、连接、构建、运行等高频命令。

## 使用方式

补全窗口会显示“命令 + 简短说明”，例如：

```text
systemctl status <服务名>    查看服务状态
systemctl start <服务名>     启动服务
Get-Service <服务名>         查看 Windows 服务状态
winget search <软件名>       搜索可安装软件包
```

插入到终端时只会插入命令本体，不会把后面的说明一起插进去。

## 命令管理

桌面端启动后，在系统托盘菜单打开：

```text
管理命令缓存
```

管理页支持：

- 按分类查看命令。
- 搜索命令、说明、标签。
- 手动新增或覆盖用户命令。
- 删除用户命令。
- 复制命令。
- 粘贴文档内容或填写官网 URL，让中转站提取命令并导入用户缓存。

也可以直接用 CLI：

```powershell
node .\bin\taih.js commands list --query systemctl
node .\bin\taih.js commands add --tool custom --command "mytool status <名称>" --summary "查看工具状态"
node .\bin\taih.js commands delete --command "mytool status <名称>"
node .\bin\taih.js commands import --from-file .\docs\tool.txt --url "https://example.com/docs"
```

## 数据规则

每条命令使用统一结构：

```json
{
  "tool": "k8s",
  "command": "kubectl get svc -A",
  "summary": "查看所有命名空间的 Service",
  "aliases": ["kube get svc -A"],
  "tags": ["kubernetes", "service", "list"],
  "source": "kubernetes-kubectl",
  "sourceUrl": ""
}
```

占位符统一使用中文尖括号，例如 `<服务名>`、`<命名空间>`、`<用户名>`。
